## 1. Purpose

This Security Awareness Program establishes the framework for ensuring all MilkyHoop personnel understand their security responsibilities and are equipped to protect organizational and tenant data.

## 2. Scope

This program applies to:
- All employees (full-time and part-time)
- Contractors and consultants
- Third-party service providers with system access
- New hires during onboarding

## 3. Program Components

### 3.1 Security Training Schedule

| Training Type | Audience | Frequency | Duration |
|--------------|----------|-----------|----------|
| New Hire Orientation | All new staff | At hiring | 2 hours |
| Annual Refresher | All staff | Annually | 1 hour |
| Developer Security | Dev team | Quarterly | 2 hours |
| Incident Response | IRT members | Semi-annually | 3 hours |
| Management Briefing | Leadership | Quarterly | 30 mins |

### 3.2 Training Topics

#### Core Topics (All Staff)
1. **Password Security**
   - Strong password creation
   - Multi-factor authentication usage
   - Password manager adoption

2. **Phishing Awareness**
   - Identifying suspicious emails
   - Reporting procedures
   - Social engineering tactics

3. **Data Handling**
   - Data classification (Public, Internal, Confidential)
   - Tenant data isolation importance
   - Secure data disposal

4. **Incident Reporting**
   - What constitutes an incident
   - Reporting channels
   - Escalation procedures

5. **Physical Security**
   - Clean desk policy
   - Visitor management
   - Device security

#### Developer-Specific Topics
1. OWASP Top 10 vulnerabilities
2. Secure coding practices
3. Secret management (SOPS usage)
4. Code review security checklist
5. Dependency vulnerability management

### 3.3 Delivery Methods

| Method | Description | Use Case |
|--------|-------------|----------|
| Online Modules | Self-paced e-learning | Core training |
| Live Sessions | Interactive workshops | New topics, incidents |
| Documentation | Written policies/guides | Reference material |
| Simulations | Phishing tests (when implemented) | Practical assessment |
| Quick Tips | Weekly security tips | Ongoing awareness |

## 4. Phishing Simulation Program

### 4.1 Implementation Plan

**Phase 1: Baseline (Q1 2026)**
- Deploy phishing simulation tool (Gophish or cloud alternative)
- Run baseline test without prior notice
- Establish click-rate baseline

**Phase 2: Education (Q2 2026)**
- Provide targeted training based on results
- Increase simulation complexity
- Track improvement

**Phase 3: Continuous (Q3+ 2026)**
- Monthly simulations with varied scenarios
- Recognize top performers
- Additional training for repeat clickers

### 4.2 Simulation Scenarios

| Difficulty | Example Scenario | Target |
|------------|------------------|--------|
| Easy | Generic "verify account" | All staff |
| Medium | Fake internal announcement | All staff |
| Hard | Targeted spear-phishing | Key personnel |

### 4.3 Metrics

| Metric | Target | Current |
|--------|--------|---------|
| Phishing click rate | < 5% | Baseline TBD |
| Report rate | > 70% | Baseline TBD |
| Training completion | 100% | 100% |

## 6. Security Champions Program

### 6.1 Purpose
Designate security-minded individuals in each team to promote security culture.

### 6.2 Responsibilities
- Act as first point of contact for security questions
- Review team's security practices
- Attend monthly security briefings
- Escalate potential issues

### 6.3 Recognition
- Quarterly recognition for top champions
- Professional development opportunities
- Badge/certification

## 8. Measurement and Improvement

### 8.1 Key Performance Indicators

| KPI | Target | Measurement |
|-----|--------|-------------|
| Training completion | 100% | Annual audit |
| Phishing resistance | < 5% click rate | Simulation results |
| Incident reporting | > 80% within SLA | Incident log |
| Security quiz pass rate | > 90% | Post-training quiz |

### 8.2 Program Review

This program is reviewed:
- Annually at minimum
- After significant security incidents
- When new threats emerge
- Upon regulatory changes

## 9. Enforcement

Non-compliance with security training requirements may result in:
- Access restrictions until training completed
- Performance review impact
- Disciplinary action for repeated violations
